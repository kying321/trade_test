## 5. 对外口径红线

所有对外材料必须继续遵守：

1. 严格区分：
   - 已验证
   - 验证中
   - 规划中
2. 不写收益承诺
3. 不写虚构客户、收入、ARR、估值
4. 不把 OI / orderflow / triad sidecar 写成成熟主执行信号
5. 不把 execution bridge 写成成熟规模化 execution SaaS
6. 不把当前状态表述成“稳定接管大规模 live capital”

当前统一主叙事：

> Fenlie 正在构建一套面向量化团队的控制面基础设施，把研究、风控、运营控制与执行接管收敛到同一条可验证、可审计的系统链中。

## 7. 版本治理规则

### 文件命名

- 统一使用：`YYYY-MM-DD-fenlie-<topic>.md`
- 不在本目录放无日期、无主题、无口径归属的临时文档

### 变更顺序

修改 investor 材料时，优先顺序固定为：

1. 先改 source 文稿
2. 再改索引与发送包
3. 最后做一致性校验

### 最低校验要求

每次新增/改动后至少校验：

1. 新文件是否被 `investor-pack-index` 收录
2. 是否破坏 `$2.2M–$2.8M / 18–24 个月` 统一口径
3. 是否误写 execution 成熟度
4. `diligence-readiness-checklist` 是否需要同步
