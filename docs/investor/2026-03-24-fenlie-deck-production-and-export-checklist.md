## 3. 成品类型

建议至少产出 3 类成品：

### A. 路演正式版

- 时长：10–15 分钟
- 页数：10–14 页
- 形式：`pptx + pdf`

### B. 会前预读版

- 时长：5 分钟内读完
- 页数：6–8 页
- 形式：`pdf`

### C. 会后补发版

- 内容较正式版多 1–2 页 appendix
- 包含 execution maturity 边界提醒
- 形式：`pdf`

---

## 4. 生产顺序

固定顺序建议为：

1. 先锁定路演正式版
2. 再从正式版裁剪出会前预读版
3. 再补会后 follow-up 版
4. 最后导出主 deck / preread / followup 三类成品

不要反过来先做会前版再硬扩成正式版，这样最容易让主叙事失真。

---

## 5. 每页最低验收要求

每一页至少检查：

1. 是否只表达一个核心观点
2. 是否仍与主叙事一致
3. 是否把 execution 说过头
4. 是否有未经验证的数字
5. 是否出现“收益导向海报化”
6. 是否能在 10 秒内看懂主标题

---

## 6. 路演正式版建议页序

1. 封面：Fenlie / 面向量化团队的控制面基础设施
2. 问题：研究、控制、handoff、execution 边界长期割裂
3. 解决方案：Research OS + Operator Console + Guarded Execution Bridge
4. 为什么现在：算法交易基础设施与数字资产机构化窗口
5. 产品架构：source-owned control chain
6. 当前最成熟能力：研究治理与控制面
7. execution 边界：验证中，不夸大
8. 商业模式：高价值交付 -> 模块化复用
9. pilot 客户画像与落地方式
10. 12–18 个月路线图
11. 融资区间与资金用途
12. 收尾：为什么现在值得见第二次
