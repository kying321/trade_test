# Fenlie Executive Summary（一页版）

## 项目一句话

Fenlie 是一套面向量化团队的反脆弱交易操作系统，目标是把 **研究、风控、运营控制与执行接管** 收敛到同一条可验证、可追溯、可部署的系统链中。

## 我们解决的问题

当前量化团队普遍存在四类断裂：

1. 研究想法、回测、参数与结论分散，难复现、难继承；
2. 风控只停留在执行层，没有贯穿研究、上线、回退与审计；
3. 执行接管缺乏统一 readiness、门禁与回退链；
4. 控制台往往只是展示层，无法作为真实运营控制面。

Fenlie 的目标不是单点优化某个因子，而是把这四类断裂收口成一套可运营的系统。

## 产品结构

Fenlie 当前由三层核心能力构成：

- **Research OS**：研究主线、回测、review、source-owned 工件链
- **Operator Console**：overview / workspace / terminal / alignment 控制台
- **Guarded Execution Bridge**：probe、ready-check、canary、takeover 的受控执行桥

## 当前阶段

### 已验证

- Crypto-first 研究链与 review 工件主线
- 面向研究与运营的控制台与公开验收链
- source-owned 状态管理与发布前 release-checklist
- OI sidecar 已接入研究增强链

### 验证中

- execution bridge 的产品化与 pilot 化
- triad 订单流增强从 context-only 向更高信任级别演进

### 规划中

- 私有化交付模板标准化
- 模块化授权与多客户部署
- 从 crypto-first 向研究型资管技术组扩张

## 为什么先从 Crypto 量化团队切入

- 当前证据链与产品成熟度最匹配 crypto 场景
- 该类团队研究与执行迭代快，对控制面需求更强
- 更适合通过 pilot、私有部署和定制交付快速形成产品反馈

## 商业模式

早期商业模式以高价值交付为主：

- 私有化部署
- 研究链与控制台落地
- 执行桥 pilot 集成
- 研究赋能与流程梳理

中长期再演进为模块化授权与平台订阅。

## 为什么值得投

Fenlie 的壁垒不在单个策略，而在于：

- 八层架构与职责隔离
- source-owned artifact discipline
- 研究—控制台—执行桥的一体化链路
- 可验收、可回归、可回退的工程化产品方法

如果未来量化竞争的核心不只是“谁找到了策略”，而是“谁能把策略装进一台更稳、更可审计的机器里”，Fenlie 构建的就是这台机器。

## 当前最适合的合作对象

- 理解量化基础设施价值的金融/量化投资人
- 愿意支持 crypto-first 楔子市场、再向机构控制面扩张的资金
- 接受“先形成强系统，再形成强平台”的长期路线
