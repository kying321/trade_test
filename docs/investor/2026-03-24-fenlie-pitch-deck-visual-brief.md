# Fenlie Pitch Deck Visual Brief

> 版本：2026-03-24  
> 用途：把 `/Users/jokenrobot/Downloads/Folders/fenlie/docs/investor/2026-03-24-fenlie-pitch-deck-full-copy.md` 转成可交给设计师、创始人本人或 AI 制图工具的页面级制作说明。  
> 适用：融资路演主 deck（12 页）+ backup（4 页）  
> 重要原则：**不做收益承诺、不伪造客户 logo、不伪造产品成熟度、不把验证中 execution 能力包装为成熟商用 SaaS。**

---

## 1. 总体视觉原则

### 1.1 设计目标

Fenlie 的 deck 不是消费互联网产品介绍，也不是“高收益策略展示页”。

视觉目标应统一为：

- **机构级**：像基础设施/控制系统，不像“量化喊单”
- **高密度但可呼吸**：一页只讲一件事，但可以有 2–3 层信息
- **克制**：减少装饰、减少无意义 3D、减少炫技动画
- **可审计**：每个关键数字都能回答“来源是什么”
- **边界清晰**：已验证 / 验证中 / 规划中 三态必须在设计上区分

### 1.2 推荐视觉风格

- 背景：深蓝黑 / 石墨灰，不用纯黑大底
- 主色：冷蓝、青蓝、少量白
- 辅助色：
  - 已验证：冷绿
  - 验证中：琥珀黄
  - 规划中：冷灰
  - 风险提醒：低饱和红
- 字体：无衬线、偏窄、工程感强
- 图标：线框、几何、少量使用，不堆图标
- 形状语言：卡片、连接线、分层矩形、状态标签、轻量网格

### 1.3 全局排版规则

每页建议采用以下层次：

1. **顶部 15%**：页标题 + 一句定位
2. **中部 55–65%**：单一主图 / 主结构
3. **底部 20–25%**：3–4 个支撑 bullet / 来源 / maturity 注记

不要出现：

- 一页 10 个以上 bullet
- 满屏小字说明
- 过多 logo 墙
- 产品页堆满所有模块
- 拿 K 线图充当产品图

### 1.4 三态视觉契约

所有页都要严格区分：

- **已验证**：实线边框、低饱和冷绿 badge、可附来源标记
- **验证中**：虚线或半透明边框、琥珀 badge、带“pilot / validation”字样
- **规划中**：灰色 outline、低对比度，不占视觉中心

推荐 badge 文案统一：

- `已验证`
- `验证中`
- `规划中`

### 1.5 不能出现的内容

严禁：

- “年化收益 xx%” 类型图
- PnL 曲线作为主卖点
- 虚构客户 logo、虚构收入、虚构 ARR
- 把 triad / orderflow research sidecar 画成成熟主执行信号
- 把 execution bridge 画成“已稳定接管所有资金”的成熟产品

---

## 2. 讲述节奏建议

整套主 deck 的讲述顺序应该是：

1. **先定义组织级问题**
2. **再给出 Fenlie 的系统级答案**
3. **再用场景解释 Research / Console / Execution Bridge**
4. **再讲证据、市场切入、商业模式、融资用途**

口径重点：

- 不先讲策略收益
- 不先讲 execution
- 不先讲 AI
- 先讲“为什么量化团队需要一套控制面系统”

## 3. 页面级视觉说明（12 页主 deck）

---

### Slide 1｜标题页

**页面目标**  
让投资人第一秒知道这不是“策略展示”，而是“量化团队控制面基础设施”。

**推荐版式**

- 左上：Fenlie wordmark
- 中部偏左：主标题 + 副标题
- 右下：三层产品结构短标签
- 底部：日期 / round discussion / crypto-first 标记

**推荐图形**

- 不需要复杂图
- 可用一张极简的“三层堆叠结构图”：
  - Research OS
  - Operator Console
  - Guarded Execution Bridge

**视觉重点**

- “控制链 / infrastructure / operating system” 的质感
- 不要用 K 线、币圈 logo、发光地球之类陈词滥调

**建议页脚元素**

- Crypto-first
- Research / Control / Execution
- 2026-03

**演讲提示**

一句话定调：Fenlie 是量化团队的操作系统，不是单一策略。

---

### Slide 2｜问题页：链路断裂

**页面目标**  
让投资人先认同“量化团队真正的痛点是链路断裂，而不是缺少一个因子”。

**推荐版式**

- 左侧：4 个断裂点
- 右侧：断裂后果
- 中间用一条被切断的流程链连接

**推荐图形**

用“Broken Operating Chain” 图示：

- Research
- Risk
- Execution
- Operations / Audit

中间用断裂、虚线、红色 warning 节点标示“state lost / no source-of-truth / no rollback discipline”。

**不建议**

- 纯 bullet 墙
- 把问题写成“找不到 alpha”

**视觉重点**

- 不是工具不够，而是系统链路不完整
- 用 1 个大图 + 4 个短块，而不是 9 行解释文字

**演讲提示**

这一页要帮投资人完成认知切换：Fenlie 卖的是系统连续性。

---

### Slide 3｜Why Now

**页面目标**  
证明这不是伪需求，而是一个随算法交易与数字资产机构化共同增长的基础设施机会。

**推荐版式**

- 上半：3 个市场信号卡
- 下半：1 句结论

**推荐图形**

优先用 **3 张数字信号卡**，不要做拥挤折线图：

1. 算法交易市场规模 / 增速
2. 传统基金数字资产参与度
3. 机构扩展到 stablecoin / tokenization 的意向

**数据来源**

- Grand View Research
- PwC + AIMA
- Coinbase Institutional + EY-Parthenon

**设计要求**

- 每张卡只放一个主数字 + 一句解释
- 来源放底部小字
- 不要把外部市场数据画成 Fenlie 自己的 traction

**演讲提示**

强调“需求成熟”，而不是“我们有一个巨大 TAM 所以必然成功”。

---

### Slide 4｜解决方案页

**页面目标**  
把 Fenlie 的产品结构一次讲清。

**推荐版式**

- 页面中间：单一大结构图
- 底部：3 个层级的定义卡

**推荐图形**

使用横向或纵向的 **三层架构图**：

`Research OS -> Operator Console -> Guarded Execution Bridge`

每层附一个 5–7 字短注释：

- 研究治理
- 运营控制
- 受控接管

**加强元素**

- 每层右上角加 maturity badge
- 执行桥明确标 `验证中`

**不建议**

- 一次展开八层内部模块细节
- 一页把所有页面截图都堆上去

**演讲提示**

这是最核心的一页，必须干净。投资人要一眼看懂 Fenlie 的产品边界。

---

### Slide 5｜场景一：研究链

**页面目标**  
说明 Fenlie 解决的不是“有没有研究”，而是“研究能不能成为团队资产”。

**推荐版式**

- 左：研究生命周期图
- 右：当前主线与纪律示例

**推荐图形**

可用一张 **Research Governance Flow**：

`idea -> backtest -> review -> canonical anchor -> next priority`

在旁边挂出 4 个标签：

- canonical anchor
- watch-only sidecar
- blocked promotion
- next research priority

**必须体现的当前事实**

- ETHUSDT
- 15m
- breakout-pullback
- 当前重点：exit/risk

**不建议**

- 放复杂收益曲线作为主图
- 放太多参数网格图

**演讲提示**

你卖的是“研究治理系统”，不是一张看起来很厉害的回测图。

---

### Slide 6｜场景二：控制台

**页面目标**  
说明 dashboard 不是展示壳，而是运营控制面。

**推荐版式**

- 左：Fenlie 控制台截图（建议 overview + workspace 组合）
- 右：4 个控制面能力标签

**推荐图形**

优先使用 **真实产品截图 + 注释 callout**：

- source-head 可见
- public / internal 分层
- artifacts / contracts / raw / alignment 可穿透
- release-checklist / acceptance chain

**截图要求**

- 截图必须真实来自现有页面
- 如需裁剪，只做信息聚焦，不重绘不存在的功能
- 尽量选深色主题一致视觉

**不建议**

- 做一张“假 UI 设计稿”冒充现有产品

**演讲提示**

投资人看到真实截图会立刻把项目从“故事”提升到“产品”。

---

### Slide 7｜场景三：执行桥

**页面目标**  
讲清 execution 是 Fenlie 的重要组成，但当前仍是验证中能力。

**推荐版式**

- 中部：受控执行桥流程图
- 底部：边界说明条

**推荐图形**

做成 **Gated Workflow Funnel**：

`probe -> ready-check -> canary -> takeover -> rollback`

每个节点下方只写 1 个关键词：

- probe：小流量验证
- ready-check：门禁确认
- canary：受控试运行
- takeover：接管
- rollback：回退

**必须出现的提醒**

- 页面显著位置放 `验证中`
- 一句固定注记：
  - “当前适合对外表述为验证型 bridge / pilot 能力，不按成熟商用 execution SaaS 表述”

**演讲提示**

这里主动降温，会提升可信度，而不是减分。

---

### Slide 8｜差异化与壁垒

**页面目标**  
告诉投资人：壁垒不在单一功能，而在系统纪律。

**推荐版式**

- 2x2 矩阵或四层壁垒塔
- 底部 1 行结论

**推荐图形**

建议使用 **Moat Stack**：

1. 八层架构与职责隔离
2. source-owned artifact discipline
3. 研究—控制台—执行桥一体化链路
4. 验收 / 回归 / 回退方法

**设计建议**

- 每一层只放一个关键词 + 一句解释
- 可以在右下角加一列“不是我们”的反向定义：
  - 不只是回测引擎
  - 不只是策略社区
  - 不只是 dashboard

**演讲提示**

这一页的任务是建立“复制难度”，不是展开全部竞品表格。

---

### Slide 9｜进展与证据

**页面目标**  
证明 Fenlie 已经从想法进入工程资产状态。

**推荐版式**

- 左：证据栈 / artifact board
- 右：3 个关键数字 tile

**推荐图形**

主图可用 **Evidence Board**：

- research mainline
- OI sidecar integration
- dashboard acceptance chain
- release-checklist gate

右侧放 3 个数字 tile：

- `24 files / 125 tests passed`
- `53` dashboard Python contracts passed
- `447 / 447 / failed=0`

**要求**

- 每个数字旁边可加“来自内部验证记录 / regression artifact”注记
- 不要把测试数说成收入 traction

**演讲提示**

这一页的核心不是“量多”，而是“工程可信度真实存在”。

---

### Slide 10｜市场切入与 GTM

**页面目标**  
解释为什么从 crypto-first 切入是正确的，不是局限。

**推荐版式**

- 左：楔子市场 ladder
- 右：客户痛点与交付方式

**推荐图形**

做一个 **3-step Expansion Ladder**：

1. Crypto-native quant team
2. Research-driven digital asset fund / tech group
3. 更广义机构控制面 / 多资产场景

旁边配一个“早期 go-to-market 方法”框：

- private deployment
- pilot integration
- enablement / advisory

**不建议**

- 一开始画全球市场全覆盖地图
- 暗示马上做横向 SaaS 爆发式扩张

**演讲提示**

顺序比规模更重要。先说明 wedge，后说明扩张。

---

### Slide 11｜商业模式与三年模型

**页面目标**  
把“高价值交付 -> 模块化授权”的演进路径讲清楚。

**推荐版式**

- 左：收入结构堆叠柱
- 右：Year 1–3 简化模型 + round discussion

**推荐图形**

推荐两块图：

1. **收入结构演进图**
   - 私有化部署
   - 研究链 / 控制台落地
   - execution bridge pilot
   - enablement / advisory
   - 后续授权 / 订阅占比抬升

2. **三年基准模型简图**
   - Year 1：$0.48M
   - Year 2：$1.42M
   - Year 3：$3.32M

**融资信息展示**

- Base round discussion：`$2.2M–$2.8M`
- runway：`18–24 个月`

**设计提醒**

- 图上要写“内部规划假设 / 非业绩承诺”
- 不要做过度陡峭的 hockey-stick

**演讲提示**

投资人宁愿看保守模型，也不想看明显吹出来的曲线。

---

### Slide 12｜Why Invest Now / Closing

**页面目标**  
完成收束：Fenlie 值得投，因为它在补量化团队真正缺的那一层。

**推荐版式**

- 左：4 个价值主张块
- 右：一句 closing statement

**推荐图形**

不必复杂。可做成 **Closing Thesis Panel**：

- 把研究变成组织资产
- 把控制台变成运营面
- 把执行接管变成受控流程
- 把验收/回退变成标准能力

右侧放最终一句：

> 如果未来量化竞争拼的是“谁能把策略装进一台更稳、更可审计的机器里”，Fenlie 正在构建的就是这台机器。

**演讲提示**

结尾不要回到局部 feature，要站在系统层和组织层结束。

## 4. Backup 页面视觉说明（4 页）

### Backup 1｜TAM / SAM / SOM

**推荐图形**

- 上层：公开市场规模参考
- 中层：Fenlie 相关基础设施子层
- 下层：3 年 SOM 目标

优先用 **三层漏斗图** 或 **收敛矩形图**，不要用饼图。

### Backup 1B｜融资方案与 runway

**推荐图形**

- 左：Lean / Base / Accelerate 三档对比表
- 右：Base 用款结构 donut 或水平条形图

视觉中心必须给 `Base（推荐）`。

### Backup 2｜竞品关系

**推荐图形**

做 **能力空白带地图**，不要做冗长竞品矩阵：

- QuantConnect：算法引擎
- FMZ：策略运行 / 策略池
- TradingView：脚本社区 / idea discovery
- Fenlie：research governance + operator control plane + guarded execution bridge

### Backup 3｜Execution 成熟度边界

**推荐图形**

用 **Maturity Ladder**：

- 工程路径已建立
- pilot / validation 适用
- 产品化治理推进中
- 尚非成熟大规模商业化 execution SaaS

### Backup 4｜研究主线与 sidecar 边界

**推荐图形**

建议做 **主线 / sidecar 分层图**：

- canonical mainline：ETHUSDT / 15m / breakout-pullback / exit-risk
- research enhancement：OI sidecar
- context-only：triad

这样能非常清楚地展示 Fenlie 的研究纪律。

## 5. 设计师/制图执行清单

### 5.1 必备素材

建议先准备：

1. 产品 logo / wordmark
2. 最新控制台真实截图
3. 一页版核心文案
4. 12+4 页完整 copy
5. 市场数据来源链接
6. 三年模型数字表
7. 融资用途占比

### 5.2 优先出图顺序

建议先做：

1. Slide 4（解决方案）
2. Slide 2（问题）
3. Slide 6（控制台）
4. Slide 7（执行桥）
5. Slide 9（证据）
6. 再补市场 / GTM / 商业模式 / closing

原因：

- 这五页最能决定整套 deck 的质感是否成立
- 做好后，其他页只需沿用视觉系统

### 5.3 演示版与发送版差异

#### 演示版

- 字更少
- 图更大
- speaker note 补充更多口播内容

#### 发送版 PDF

- 每页可多一行解释
- 来源标记更完整
- 需要能在没人讲解时也自解释

## 6. 推荐制作工具路径

### 方案 A：Keynote / Figma / PPT 手工精修

适合正式融资会面。

### 方案 B：先用 AI 生成图示，再人工二次精修

适合快速成稿，但必须人工审阅成熟度表述与图形真实性。

### 方案 C：先做文字版 PDF，再升级设计版

适合先约会、后逐步完善设计的节奏。

当前推荐顺序：

1. 先用本文件做结构稿
2. 生成 12 页粗版
3. 创始人口播走一遍
4. 修正“太虚 / 太满 / 太早”的页
5. 再做细节美化

## 7. 最后审查清单

路演前逐页检查：

- [ ] 是否把 execution bridge 明确标为“验证中”
- [ ] 是否没有任何收益承诺或夸张 PnL 表达
- [ ] 是否没有虚构客户、收入、ARR、logo
- [ ] 是否所有数字都能说出来源
- [ ] 是否第一页就把 Fenlie 讲成控制面基础设施，而不是策略页
- [ ] 是否控制台截图为真实页面
- [ ] 是否主 deck 12 页以内就能讲清故事
- [ ] 是否 backup 页只做支撑，不喧宾夺主

## 8. 一句话制作标准

> 一套好的 Fenlie 融资 deck，不是把功能堆满，而是让投资人看见：这是一套正在成形的量化团队控制面系统，而且它知道自己的边界。
