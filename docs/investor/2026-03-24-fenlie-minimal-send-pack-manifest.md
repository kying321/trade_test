# Fenlie Minimal Send Pack Manifest

> 版本：2026-03-24  
> 用途：给创始人、运营或顾问一个可直接执行的最小发件清单，避免每次临时挑文件、混发错误版本或把尽调材料过早全部发出。  
> change class：`DOC_ONLY`

---

## 1. 一句话原则

对外发件必须分层，不允许“第一次接触就把整个 investor 目录全发出去”。

最小发件原则是：

> **先建立认知，再建立叙事，再进入模型与尽调。**

## 3. 推荐发件顺序

固定顺序：

1. `Pack A`
2. `Pack B`
3. `Pack C`
4. `Pack D`

不建议跳跃：

- 不建议 `A -> D`
- 不建议 `A -> 全部目录`
- 不建议在尚未会面时直接发 execution / diligence 全套

---

## 4. 附件数量上限

为了降低对方阅读负担，建议单次发件控制为：

- 首次触达：**2–3 份**
- 会前预读：**3–4 份**
- 会后推进：**4–5 份**
- 尽调入口：**不超过 7 份**，并优先给 index 而不是全量正文

---

## 5. 标准邮件主题建议

### 首次触达

- `Fenlie / 量化团队控制面基础设施 / 简要材料`

### 会前预读

- `Fenlie 会前预读材料`

### 会后推进

- `Fenlie 会后补充：商业模式、融资与 execution 边界`

### 尽调入口

- `Fenlie diligence / pilot material index`

---

## 6. 红线

绝不在发件包中混入：

1. 未审计经营数字
2. 虚构客户 / 收入 / ARR / 估值
3. 收益承诺
4. 把 execution bridge 说成成熟规模化 execution SaaS
5. 把 sidecar 研究增强项说成当前成熟主执行信号
6. 多个互相冲突版本的同类材料

---

## 7. 发件前检查清单

- [ ] 当前附件是否都在 `investor-pack-index` 中
- [ ] `$2.2M–$2.8M / 18–24 个月` 融资口径是否一致
- [ ] execution 是否仍按“验证中能力”表述
- [ ] 是否错误混入尽调过深材料
- [ ] 是否存在更晚版本但未替换
